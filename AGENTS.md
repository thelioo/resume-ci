# Agent Instructions

## Project Shape

This is a **GitHub template repository** for building LaTeX/PDF resumes from YAML data files.

Users fork this repo, add their YAML resume data, push to `main`, and GitHub Actions generates the PDF as a workflow artifact. No local LaTeX or Python installation required.

It is not a LinkedIn automation project, job applier, recruiter response generator, interview prep system, or general career knowledge base.

## Hard Rules

### 1. Never Edit Generated `.tex` Files

Do not edit files in `data/output/latex/*.tex` directly. They are generated by the build pipeline.

Edit only the YAML source files:

```
data/output/latex/resume-data.yml      (or resume-data-<lang>.yml)
```

Then run or trigger CI:

```bash
python3 skills/resume-manager/scripts/build_resume.py \
  --template templates/output/latex/curriculo_template.tex
```

### 2. Templates Are for Layout Only

Do not modify `templates/output/latex/curriculo_template.tex` unless the user explicitly asks for template or layout changes.

For content changes, edit the YAML only.

### 3. Keep Personal Data Out Of Git

`data/output/latex/*.yml` files contain personal information. The repository should be kept private. Do not force-add `data/input/` or `profile/`.

### 4. Use Plain ASCII in Resume Content

Generated resume content must avoid emojis, decorative unicode, fancy quotes, em dashes, and en dashes. Prefer plain punctuation and standard ASCII.

## Architecture

```
data/output/latex/resume-data*.yml       → source data (user-edited)
templates/output/latex/curriculo_template.tex  → LaTeX template (layout)
skills/resume-manager/scripts/build_resume.py  → validator + renderer + compiler
data/output/latex/[name].tex             → generated (never edit)
data/output/latex/[name].pdf             → generated (CI artifact)
```

## CI Pipeline

`.github/workflows/build-resume.yml` triggers on push to `main` when YAML, template, or script files change. It installs LaTeX via `apt-get`, runs the build script, and uploads PDFs as workflow artifacts (90-day retention).

## Workflow

1. Read `data/output/latex/resume-data*.yml`.
2. Make the smallest correct YAML change.
3. Run `python3 skills/resume-manager/scripts/build_resume.py --template templates/output/latex/curriculo_template.tex`.
4. Report the generated PDF path.

## Resume Data Schema

```yaml
personal:
  name: string
  title: string
  email: string
  linkedin_url: string
  github_url: string

font: lmodern | charter | cormorant | fira-sans | source-sans

section_titles:
  experience: string
  projects: string
  skills: string
  education: string

experience:
  - company: string
    period:
      from: string
      to: string
    role: string
    url: string
    bullets:
      - string

projects:
  - company: string
    period:
      from: string
      to: string
    role: string
    url: string
    bullets:
      - string

skills:
  - label: string
    items: string  # comma-separated

education:
  - institution: string
    period:
      from: string
      to: string
    degree: string
    location: string

output_filename: string  # letters, digits, _ and - only
```

`projects: []` hides the projects section.

## Bullet Formatting

Use rich text markers in YAML bullets:

| Marker | Rendered as |
|---|---|
| `**text**` | `\textbf{text}` |
| `_text_` | `\textit{text}` |

Do not write raw LaTeX formatting commands inside YAML bullets.
