# Agent Instructions for OpenCode

## Project Scope

This project is now only a resume manager.

It is not a LinkedIn automation project, job applier, recruiter response generator, interview prep system, or general career knowledge base.

## Hard Rules

### 1. Never Edit Generated `.tex`

Do not edit files in `data/output/latex/*.tex` directly. They are generated by the build pipeline.

Edit only:

```txt
data/output/latex/resume-data.yml
```

Then run:

```bash
python3 scripts/build_resume.py
```

### 2. Templates Are Structural References

Do not modify `templates/output/latex/curriculo_template.tex` unless the user explicitly asks for template/layout changes.

For content changes, edit the YAML only.

### 3. Keep Personal Data Out Of Git

`data/` contains personal data and generated outputs. Do not force-add it unless the user explicitly asks and understands the privacy risk.

### 4. Use Plain Characters In Resume Content

Generated resume content should avoid emojis, decorative unicode, fancy quotes, em dashes, and en dashes. Prefer plain punctuation and standard ASCII where possible.

## Architecture

```txt
data/output/latex/resume-data.yml    -> source data
templates/output/latex/              -> LaTeX template
scripts/build_resume.py              -> YAML + template -> .tex -> .pdf
data/output/latex/[name].tex         -> generated
data/output/latex/[name].pdf         -> generated
```

## Workflow

1. Read `data/output/latex/resume-data.yml`.
2. Make the smallest correct YAML change.
3. Run `python3 scripts/build_resume.py`.
4. Report the generated PDF path.

## Resume Data Schema

```yaml
personal:
  name: string
  title: string
  email: string
  linkedin_url: string
  github_url: string

experience:
  - company: string
    period:
      from: string
      to: string
    role: string
    url: string
    bullets:
      - string

projects:
  - company: string
    period:
      from: string
      to: string
    role: string
    url: string
    bullets:
      - string

skills:
  - label: string
    items: string

education:
  - institution: string
    period:
      from: string
      to: string
    degree: string
    location: string

output_filename: string
```

`projects: []` hides the projects section.

## Bullet Formatting

Use rich text markers in YAML:

| Marker | Rendered as |
|---|---|
| `**text**` | `\textbf{text}` |
| `_text_` | `\textit{text}` |

Do not write raw LaTeX formatting commands inside YAML bullets.
